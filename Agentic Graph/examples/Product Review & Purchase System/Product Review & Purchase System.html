Product Review & Purchase System

This project contains 4 agents, 11 tools, 3 knowledge sources.

Project Details

Description

Multi-agent e-commerce assistant that helps customers discover products through reviews, check availability and pricing, and complete purchases with delivery tracking.

Business Justification

Reduce customer support load by 40% and increase conversion rate through guided product discovery and frictionless checkout.

  • Target Completion: Q2 2026
  • Estimated Effort: 8 weeks (2 engineers)
  • Team Size: 2 engineers + 1 PM
  • Integration Points: Stripe (payments), AWS OpenSearch (reviews), ERP system (inventory), CRM (customer profiles)
  • Deployment Target: Cloud
  • Overall Risk: Medium
  • Compliance: PCI-DSS for payment processing, GDPR/CCPA for customer data
  • Data Classification: Contains PII (customer profiles) and financial data (payment tokens)
  • Regulatory Constraints: Payment processing must be PCI-DSS Level 1 compliant

Critical Dependencies

Stripe API availability, warehouse inventory sync latency

Key Assumptions

Customers have existing accounts with saved addresses; products are physical goods with warehouse fulfilment

Open Questions / Blockers

Should the Purchase Agent handle returns/refunds? Digital goods support needed in v2?

Purchase Agent

Handles the checkout flow: validates delivery address, processes payment, confirms the order, and provides a tracking number.

  • Framework: Custom

Tool

Review Search API

Searches the product review index by SKU or keyword. Returns up to 20 reviews with ratings, text, and verified-purchase flag.

  • Type: API
  • Execution: Sync
  • Inputs: product_sku: String, limit: Int = 20, sort_by: relevance | date
  • Outputs: reviews: [{rating, text, author, date, verified}]
  • Auth Method: API Key
  • Endpoint: https://api.store.com/v2/reviews/search
  • Timeout: 5000
  • Error Handling: Retry

Inventory API

Real-time warehouse stock lookup. Returns quantity, warehouse location, price, and delivery estimate for a given SKU and destination ZIP.

  • Type: API
  • Execution: Sync
  • Inputs: product_sku: String, destination_zip: String
  • Outputs: {quantity, warehouse, unit_price, delivery_estimate}
  • Auth Method: API Key
  • Endpoint: https://api.store.com/v2/inventory/check
  • Timeout: 3000
